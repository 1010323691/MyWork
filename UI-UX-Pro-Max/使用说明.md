# UI UX Pro Max - 使用说明

## 安装方法

### 方法一：CLI（推荐）

```bash
npm install -g uipro-cli
cd /path/to/your/project
uipro init --ai claude      # 或 cursor, windsurf 等
```

### 方法二：Claude Marketplace
```bash
/plugin marketplace add nextlevelbuilder/ui-ux-pro-max-skill
/plugin install ui-ux-pro-max@ui-ux-pro-max-skill
```

### 全局安装（所有项目）
```bash
uipro init --ai claude --global   # 安装到 ~/.claude/skills/
```

---

## 使用方式

### 自动激活模式（技能模式）

技能会在请求 UI/UX 工作时自动激活。只需自然地聊天：

> "Build a landing page for my SaaS product"

**支持的工具**: Claude Code, Cursor, Windsurf, Antigravity, Codex CLI, Continue, Gemini CLI, OpenCode, Qoder, CodeBuddy, Droid, KiloCode, Warp, Augment

### Slash Command 模式（工作流模式）

使用 `/ui-ux-pro-max` 命令调用技能：
```
/ui-ux-pro-max Build a landing page for my SaaS product
```

**支持的工具**: Kiro, GitHub Copilot, Roo Code, KiloCode

## 高级用法：设计系统命令

通过 Python 脚本直接访问设计系统生成器：

```bash
# 生成带 ASCII 输出的结果
python3 .claude/skills/ui-ux-pro-max/scripts/search.py "beauty spa wellness" --design-system -p "Serenity Spa"

# 持久化到文件，用于分层检索
python3 .claude/skills/ui-ux-pro-max/scripts/search.py "SaaS dashboard" --design-system --persist -p "MyApp"
```

**持久化的结构**: 
```<span class="math inline">\</span>design-system/
├── MASTER.md           # 全局单一事实来源
└── pages/
    └── dashboard.md    # 特定页面的覆盖设置
```

## 为贡献者的架构说明

代码库使用**基于模板的生成系统**。特定于平台的文件由 CLI 动态生成。

### 目录结构：
```
src/ui-ux-pro-max/           # 真实来源（数据、脚本、模板）
cli/                         # CLI 安装器
.claude/                     # Claude Code 的本地开发/测试
.factory/                    # Droid 的本地开发/测试 (Factory)
```

### 贡献者快速入门：
1. 克隆仓库并导航到目录
2. 在 `src/ui-ux-pro-max/data/*.csv`、`scripts/*.py`或`templates/`中进行更改
3. 同步到 CLI assets
4. 构建 CLI: `cd cli && bun run build`
5. 使用 offline 标志进行本地测试
6. 创建特性分支和 PR
