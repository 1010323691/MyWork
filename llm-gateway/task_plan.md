## 已做决策
| 决策 | 理由 |
|------|------|
| 使用 `logback-spring.xml` 统一日志行为 | Spring Boot 原生支持 profile 分支，适合同时控制控制台与文件输出 |
| `prod` 采用文件日志，非 `prod` 保持控制台 | 与用户要求一致，且不影响开发体验 |
| 错误日志单独输出完整异常堆栈 | 方便排障，同时保持应用日志主体更易读 |
